# Habit Navigator — дизайн спецификация

**Дата:** 2026-07-08
**Собственик:** Крис (клиент №1 на собствената си habit платформа)
**Цел:** 30-дневен gamified dashboard за (1) изграждане на ежедневен тай-чи навик и (2) отказване от цигарите (22–23/ден → 0 на Ден 21), който едновременно генерира чист dataset за трениране на habit платформата на Крис.

---

## 1. Контекст — 30-дневният план (одобрен)

Научна основа: habit loop (cue→craving→response→reward), Four Laws (Clear), hot triggers,
среда > воля, Mini Habits („никога нула"), reward revaluation за вътрешни тригери (Brewer).

- **Тай-чи:** всяка сутрин в 6:00, веднага след миене на зъбите, ПРЕДИ кафето —
  замества цигара №1. 10 мин форма (Крис я знае наизуст). Минимум в лош ден: 2 мин.
  Награда: кафето след това + чек в dashboard-а.
- **Цигари — 3-седмичен taper с тавани:**

  | Дни | Таван/ден |
  |---|---|
  | 1–3 | 20 |
  | 4–7 | 16 |
  | 8–10 | 12 |
  | 11–14 | 8 |
  | 15–17 | 5 |
  | 18–20 | 3 |
  | **21** | **0 — Quit Day** |
  | 22–30 | 0 (консолидация; пик на отнемане Дни 23–26) |

- **Среда:** кола = 100% без цигари (Ден 1); пакетът извън кабинета на работа;
  жена му пуши — молба за 3 неща (не предлага, пакетът извън поглед, без пушене
  в тригер-моментите му). Гладове, които не могат да се дизайнират → Brewer 3 Gears.
- **Уникален лост:** всеки устоян глад = тренировъчна данна за собствения му продукт.

## 2. Архитектура

- **Frontend:** един самостоятелен `index.html` — vanilla HTML/CSS/JS, mobile-first,
  тъмна елегантна тема. Chart.js през CDN за графиките. Без build стъпка.
- **Backend:** нов Supabase проект `habit-navigator` — Postgres + Auth + RLS.
  Клиентът говори директно със supabase-js v2 (CDN ESM import).
- **Хостинг:** GitHub Pages (ново repo `habit-navigator`, main branch, root).
  На телефона: Add to Home Screen. Обосновка: версиониране на продуктов прототип,
  стабилен HTTPS URL, deploy = git push; Supabase остава само данни+auth.
- **Auth:** Supabase email+парола, един потребител (Крис), signup изключен —
  потребителят се създава при setup. Сесията се пази per устройство (login веднъж).
- **Offline устойчивост:** бързите тапове се пишат в localStorage outbox и се
  flush-ват към Supabase при връзка — тап никога не се губи и никога не бави.
- **Export:** бутон „Данни" → JSON (пълен dump на трите таблици) + CSV
  (craving_events и habit_days поотделно) — dataset за платформата.

## 3. Модел на данните (Postgres)

Всички таблици имат `user_id uuid default auth.uid()`; RLS: `auth.uid() = user_id`
за SELECT/INSERT/UPDATE/DELETE.

### 3.1 `settings` (1 ред)
| Колона | Тип | Default |
|---|---|---|
| user_id | uuid PK | auth.uid() |
| start_date | date | — (задава се при първи вход) |
| quit_day_offset | int | 21 |
| baseline_cigs | numeric | 22.5 |
| pack_price_eur | numeric | 3.60 (Marlboro Touch) |
| cigs_per_pack | int | 20 |

### 3.2 `habit_days` (1 ред/ден)
| Група | Колони |
|---|---|
| Ключ | `day date`, PK (user_id, day) |
| Тай-чи (сутрин) | `taichi_minutes int`, `taichi_quality int 1–5`, `state_before int 1–5`, `state_after int 1–5` |
| Сутрешен чекин | `sleep_quality int 1–5`, `morning_craving int 1–5` (кафе без цигара), `confidence int 1–5` (ще спазя тавана), `morning_done_at timestamptz` |
| Вечерен вход | `cig_count_final int` (reconciliation), `mood int 1–5`, `stress int 1–5`, `wife_smoked bool`, `alcohol bool`, `identity_vote int 1–5` („днес действах като непушач"), `hardest_moment text`, `what_helped text`, `withdrawal jsonb` (пост-quit чекбокси: раздразнителност/концентрация/глад_за_храна/друго), `note text`, `evening_done_at timestamptz` |
| Изчислими (не се пазят) | таван за деня (от графика), XP, статус на веригите |

### 3.3 `craving_events` (append-only — златото за системата)
| Колона | Тип / стойности |
|---|---|
| id | uuid PK |
| ts | timestamptz default now() |
| kind | `smoked` \| `resisted` |
| trigger | `стрес` \| `кафе` \| `след_ядене` \| `скука` \| `пауза_работа` \| `шофиране` \| `жена_ми_запали` \| `алкохол` \| `друго` |
| intensity | int 1–5 |
| instead | (само за resisted) `дишане_60с` \| `отложих_10мин` \| `микро_тай_чи` \| `чай_вода` \| `отмина_само` \| `друго` |
| note | text, optional |

## 4. Трите входа (UX)

### 🌅 Сутрешен чекин (~60 сек, след тай-чи)
Тай-чи минути (бърз избор: 2/5/10/15/друго) → качество 1–5 → състояние преди/след
1–5 → сън 1–5 → глад при кафето 1–5 → увереност за тавана 1–5. Един екран, тап-tap-tap.

### 📱 Бързи тапове (≤10 сек, през целия ден)
Два големи бутона на началния екран: **ИЗПУШИХ** / **УСТОЯХ**.
Тап → bottom sheet: тригер (грид от 9 чипа) → интензитет (5 точки) →
[само Устоях] какво направи вместо (6 чипа) → авто-запис, sheet се затваря.
Никакви задължителни текстови полета.

### 🌙 Вечерен вход (~3–4 мин, след 21:30)
Потвърди брой цигари (предложен от таповете, коригируем) → настроение/стрес →
среда (жена пуши/алкохол) → глас за идентичност 1–5 → най-труден момент +
какво помогна (по 1 ред) → [след Ден 21] симптоми на отнемане → бележка (optional).

## 5. Геймификация

### 5.1 Вериги (гръбнак)
- **🔥 Тай-чи верига:** ден се брои при `taichi_minutes ≥ 2` („никога нула").
- **🍃 Чиста верига:** преди Quit Day — `cig_count_final ≤ таван`; след — `= 0`.
- Верига се къса само от провален ден; издънка в рамките на тавана не къса нищо.

### 5.2 XP и нива
| Действие | XP |
|---|---|
| Тай-чи ≥ 10 мин | +20 |
| Тай-чи 2–9 мин (мини) | +10 |
| Устоян глад | +5 |
| Ден под/на тавана | +15 |
| Сутрешен чекин | +5 |
| Пълен вечерен вход | +10 |

Нива: Начинаещ 0 → Ученик 300 → Практикуващ 800 → Пазител на веригата 1500 →
Майстор на навика 2400.

### 5.3 Значки
„Първи ден" · „Чиста кола ×3" · „Седмица тай-чи" (7 дни) · „≤16" · „≤12" · „≤8" ·
„QUIT DAY" (Ден 21 с 0) · „Оцелях пика" (Дни 22–26 чисти) · „100 устояни глада" ·
„30-дневна верига" (тай-чи) · „Клиент №1" (30 пълни дневника — dataset-ът е събран).
Изчисляват се клиентски от данните; не се съхраняват.

### 5.4 Живи метрики
- Неизпушени цигари: Σ по дни `max(0, baseline − cig_count_final)`
- Спестени €: неизпушени ÷ 20 × 3.60
- Върнати часове живот: неизпушени × 11 мин
- **Медицинска линия** (тиктака от Quit Day): 20 мин пулс ↓ → 12 ч CO излиза →
  48 ч вкус/мирис ↑ → 2 седм кръвообращение ↑ → 1 мес дробове ↑ (+ preview нататък)

## 6. Екрани (tab bar: Днес / Прогрес / Постижения / Данни)

1. **Днес:** двете вериги (пламъче + брояч), „Ден X от 30", днешен таван + оставащи,
   двата грамадни тап-бутона, XP бар + ниво, CTA за дължимия вход (сутрешен/вечерен).
2. **Прогрес:** календар-heatmap 30 дни (двуцветен: тай-чи/чист) · линия цигари/ден
   срещу стъпаловидния таван · heatmap „гладове по час × ден" · бар тай-чи минути ·
   линии сън/стрес/настроение.
3. **Постижения:** значки (отключени цветни / заключени сиви + условие),
   живите метрики, медицинската линия.
4. **Данни:** export JSON/CSV, настройки (цена, baseline, start_date), logout.

## 7. Обработка на грешки
- Offline/бавна мрежа: outbox в localStorage, автоматичен retry; UI винаги
  реагира мигновено (optimistic).
- Двоен тап/дубликати: клиентски client_id на event + unique constraint.
- Пропуснат вечерен вход: на другия ден dashboard-ът предлага „попълни вчера"
  (без наказание — данни > вина; late-фактът личи от `evening_done_at`).
- Editable история: последните 48 ч. могат да се коригират; по-старото е read-only.

## 8. Тестване
- SQL: RLS политики проверени с втори тестов потребител (не вижда данните на Крис).
- Клиент: ръчен happy-path на телефон + desktop; offline тест (airplane mode тап →
  връзка → flush); проверка на XP/вериги срещу ръчно сметнат пример.

## 9. Извън обхват (YAGNI)
Множество потребители/навици, push нотификации, service worker/пълен PWA, AI анализ
в dashboard-а (това е работата на платформата — тук само събираме чист dataset),
автоматичен sync към Obsidian/Pinecone.
